Overview

This project was the completion of my Computer Engineering diploma. The idea my partner and I
came up with was making our own IOT kit. which was neat but I had an issue when designing it
with him. You see at the time we were doing a couse on sockets, but we had already learned about REST. The idea behind this project was to make several small IOT devices that you could configure to interact with eachother so for example configure a button to turn on a light. Now my partner wanted
to be safe and stick with REST for the communications between the boards and the sever this seemed
like a bad idea to me since I knew REST requests make several tcp sockets and our little ardino boards probably couldnt handel making all the connections with its limited memeory. This was hard to
relay to my partner so I took charge of sistuation and said I'd do the research and develop a server for our needs. Using python and the ardino SDK I build the client and server for our project. The
icing on the cake was seeing other projects using REST and having to deal with massive latenty like
I predicted. This project really taught me to stick to my guns and question things I dont feel
right about
